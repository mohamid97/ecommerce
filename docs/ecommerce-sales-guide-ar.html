ملخص المنتج

المنتج عبارة عن منصة API متكاملة لإدارة متجر إلكتروني متعدد اللغات. المنصة تغطي رحلة العميل من التصفح والبحث وحتى السلة والطلب، وتغطي رحلة الإدارة من إنشاء المنتجات والمخزون والخصومات وحتى متابعة الطلبات والإحصائيات.

نقطة البيع الأساسية: العميل لا يحصل على صفحة عرض منتجات فقط، بل يحصل على نظام تشغيل للمتجر: كتالوج، variants، مخزون، bundles، كوبونات، نقاط ولاء، طلبات ضيف ومستخدم، wishlist، شحن، عملاء، صلاحيات، CMS، وإحصائيات.

للعميل النهائي تصفح منتجات، فلترة وبحث، تفاصيل المنتج، اختيار variant، إضافة للسلة، wishlist، كوبونات، نقاط، وإتمام طلب كضيف أو مستخدم.
لإدارة المتجر إدارة منتجات ومخزون وطلبات وعملاء وخصومات وباقات وشحن وصلاحيات ومحتوى الموقع من مكان واحد.

كيف نشرح المنتج في مكالمة مبيعات

  • ابدأ بالقيمة: المنصة تساعد أي شركة تبيع منتجات على بناء متجر منظم، قابل للتوسع، ومناسب للغات متعددة.
  • وضح المرونة: المنتج يمكن أن يكون بسيط بدون options، أو له options/variants مثل اللون والمقاس والحجم.
  • اربطها بالعمليات: المخزون، الطلبات، المرتجعات، الخصومات، والنقاط كلها جزء من دورة تشغيل واحدة.
  • طمن العميل: الطلبات تتم داخل transaction، والمخزون يتم التحقق منه قبل السلة والطلب لتقليل overselling.
  • بيع الإدارة وليس الواجهة فقط: النظام فيه admin APIs وصلاحيات وإحصائيات ومحتوى، وبالتالي يخدم التشغيل اليومي وليس الظهور فقط.

أنواع المنتجات

النوع الشرح لفريق المبيعات مثال ما الذي يدعمه النظام
منتج بدون options منتج واحد بسعر ومخزون مباشر، لا يحتاج اختيار لون أو مقاس. كتاب، أداة واحدة، منتج ثابت. صورة، SKU، barcode، سعر، خصم، stock، حالة، تصنيف، brand، industries، مواصفات وشحن.
منتج له options/variants منتج له اختيارات، وكل اختيار نهائي يعتبر variant مستقل بسعر ومخزون وصور. T-shirt له لون ومقاس، أو منتج زراعي له حجم عبوة. Options، option values، توليد combinations، variant افتراضي، أسعار وخصومات ومخزون وصور لكل variant.
Bundle / Bundel باقة تجمع أكثر من منتج أو variant بهدف زيادة قيمة الطلب أو بيع مجموعة جاهزة. Starter kit، عرض مجموعة منتجات، باقة موسم. منتجات داخل الباقة، كميات، variants مسموحة، سعر وخصم للباقة، حالة، صورة، تصنيف وbrand.
رسالة مهمة للمبيعات: عند سؤال العميل "هل المنتج ممكن يكون له option أو لا؟" الإجابة: نعم. النظام يدعم الاثنين. لو المنتج بسيط يتم بيعه مباشرة، ولو له اختيارات يتم تفعيل `has_options` وإدارة كل variant بشكل مستقل.

Features المتاحة

1. إدارة المنتجات والكتالوج

Catalog

شرح المنتج: إدارة كاملة للمنتجات المعروضة في المتجر مع بيانات عربية وإنجليزية، صور، وصف، SEO، تصنيف، brand، ترتيب، حالة، سعر، وخصم.

القيمة البيعية: العميل يقدر ينظم كتالوج كبير ويعرضه بلغتين مع تحكم كامل في ظهور المنتجات.

  • إنشاء وتحديث وعرض وحذف المنتجات من لوحة الإدارة.
  • دعم صور رئيسية ومعرض صور وbreadcrumb image وalt/title image.
  • ربط المنتج بـ category وbrand وindustries.
  • حالات المنتج: active، draft، unavailable.
  • خصم fixed أو percentage على المنتج.
  • منتجات featured، newest، last-piece.
  • Related products لاقتراح منتجات مرتبطة.
  • Public APIs لعرض قائمة المنتجات، التفاصيل، newest، last-piece، related.

2. Options و Variants

Product Options

شرح المنتج: يمكن تعريف options عامة مثل اللون أو المقاس، وقيم لكل option. بعدها يتم ربطها بالمنتج وإنشاء variants من combinations.

القيمة البيعية: مناسب للمتاجر التي تبيع نفس المنتج بأشكال مختلفة مع أسعار ومخزون مختلف لكل اختيار.

  • Option له title مترجم، code، image، ونوع قيمة `value_type` نص أو صورة.
  • Option values مترجمة، ويمكن أن تكون قيمة نصية أو صورة swatch.
  • المنتج ذو options يتطلب product_options وقيم صالحة تابعة لنفس option.
  • إمكانية توليد combinations للـ variants.
  • كل variant له SKU، barcode، status، سعر، خصم، stock، أبعاد، وزن، مدة توصيل، وصور.
  • إمكانية تحديد default variant.
  • صور عامة للـ variants وصور خاصة لكل variant.

3. المخزون و Stock Batches

Inventory

شرح المنتج: إدارة المخزون تتم من خلال batches/stock movements، سواء لمنتج بسيط أو variant.

القيمة البيعية: يساعد العميل على التحكم في الكميات، تكلفة الشراء، وسعر البيع، مع تقليل بيع منتجات غير متاحة.

  • إضافة batch بمقدار quantity، cost price، sale price، note، status.
  • تحديث batch وعرض تفاصيله وحذفه وتغيير حالته.
  • التحقق من المخزون قبل الإضافة للسلة وتحديث الكمية وإنشاء الطلب.
  • تسجيل تخصيص المخزون داخل order item batches.
  • استرجاع المخزون عند تغيير الطلب إلى refunded.

4. السلة Cart

Cart

شرح المنتج: السلة تدعم المنتجات البسيطة، منتجات variants، والـ bundles. يوجد سلة للمستخدم المسجل ومعاينة للضيف بدون حفظ دائم.

القيمة البيعية: رحلة شراء مرنة: العميل يقدر يجرب السلة كضيف أو يكمل كمستخدم، والمتجر يحافظ على دقة الأسعار والمخزون.

  • إضافة للسلة، تحديث كمية، حذف item، حذف كل السلة، عرض السلة.
  • استراتيجية مختلفة حسب نوع العنصر: simple product، product with option، bundle.
  • حساب total_before_discount و total_after_discount لكل عنصر.
  • Guest cart preview من payload بدون تسجيل دخول.
  • Admin يمكنه عرض السلات وحذف عناصر أو تفريغ سلة مستخدم.

5. الطلبات Orders

Checkout

شرح المنتج: النظام يدعم إنشاء طلبات للمستخدم المسجل من السلة الحالية، وطلبات للضيف من قائمة items مباشرة، مع preview قبل التأكيد.

القيمة البيعية: يدعم conversion أعلى لأن العميل ليس مجبرا على التسجيل قبل الطلب، وفي نفس الوقت العضو يحصل على مزايا إضافية مثل النقاط وطلباته السابقة.

  • Order preview للمستخدم والضيف لحساب الإجمالي قبل إنشاء الطلب.
  • حقول الطلب: الاسم/الإيميل للضيف، الهاتف، المحافظة، عنوان الشحن، طريقة الدفع.
  • حساب إجمالي قبل الخصم، بعد الخصم، الشحن، الكوبون، والنقاط.
  • إنشاء الطلب داخل database transaction.
  • حفظ snapshots لعناصر الطلب والباقات والـ variants.
  • عرض قائمة طلبات المستخدم وتفاصيل الطلب برقم الطلب.
  • Admin يمكنه عرض الطلبات، تفاصيل الطلب، ملخص طلبات العميل، إنشاء طلب ضيف، وتحديث status/payment status.

6. حالة الطلب والدفع

Operations

شرح المنتج: الإدارة تتحكم في حالة الطلب وحالة الدفع، مع تأثيرات تشغيلية مثل زيادة عداد المبيعات عند التسليم واسترجاع المخزون عند الاسترداد.

  • Statuses مدعومة في الإحصائيات: pending، confirmed، processing، shipped، delivered، cancelled، refunded.
  • payment_status مثل paid/unpaid حسب البيانات الموجودة في النظام.
  • عند delivered لأول مرة يتم زيادة sales_number للمنتجات والـ variants والـ bundles.
  • عند refunded يتم رجوع المخزون للـ batches الأصلية.

7. Bundles / Bundels

Upsell

شرح المنتج: إنشاء باقات تحتوي على منتجات متعددة، مع تحديد كميات وربما variants مسموحة لكل منتج داخل الباقة.

القيمة البيعية: ميزة قوية لرفع متوسط قيمة الطلب AOV وعمل عروض جاهزة وسهلة الفهم.

  • إنشاء وتحديث وحذف وعرض الباقات.
  • صورة للباقة، title/des مترجم، category، brand، status.
  • خصم للباقة fixed أو percentage.
  • حساب سعر الباقة من مكونات المنتجات أو من خصم الباقة.
  • التحقق من أن منتجات الباقة active وأن بها variants متاحة عند الحاجة.
  • إذا منتج أو variant داخل باقة أصبح غير متاح، يمكن تحويل الباقة draft أو حذفها من السلة.

8. Promotions و Coupons

Marketing

شرح المنتج: إدارة عروض وكوبونات قابلة للربط بمنتجات أو تصنيفات أو brands أو الطلب كله، مع تواريخ صلاحية وحدود استخدام.

القيمة البيعية: فريق التسويق يقدر يطلق حملات discount و coupon codes بدون تعديل تقني كل مرة.

  • start_date و end_date، status، صورة ومكان ظهور مثل hero أو offers_section أو pop_up أو header_alert.
  • is_coupon لتحديد هل العرض كوبون قابل للإدخال في checkout.
  • coupon_code و coupon_limit.
  • نوع الخصم: percent، fixed، bundle، bulk، buy-x-get-y.
  • target: global، category، product، brand، order.
  • max_amount_discount لتحديد سقف الخصم.
  • customer_group: all، new_user، registered.

9. Wishlist

Retention

شرح المنتج: المستخدم المسجل يمكنه حفظ منتجات في wishlist، وإضافة أو حذف أو toggle.

القيمة البيعية: تزيد رجوع العميل للشراء لاحقا، وتساعد في حملات remarketing أو العروض المستقبلية.

  • Wishlist للمستخدمين المسجلين فقط.
  • add، toggle، remove، view.
  • عرض wishlist في admin للمراجعة والتحليل.
  • ترجع بيانات product أو variant مناسبة للواجهة.

10. Loyalty Points

Loyalty

شرح المنتج: نظام نقاط يسمح بإعداد tiers/conversion، استخدام نقاط لتقليل قيمة الطلب، ومنح نقاط بعد اكتمال الطلب المدفوع.

القيمة البيعية: مناسب لزيادة تكرار الشراء وتحويل العميل من مشتري مرة واحدة إلى عميل دائم.

  • Admin: عرض tiers، إضافة tier، حذف tier.
  • الطلب يدعم `use_points` و `points_to_use` للمستخدم المسجل.
  • يتحقق من الحد الأدنى لقيمة الطلب ومن رصيد المستخدم.
  • قيمة النقاط لا تتجاوز قيمة الطلب.
  • منح نقاط عند status = delivered و payment_status = paid.

11. الشحن والمحافظات

Shipping

شرح المنتج: الطلبات تعتمد على government_id وعنوان شحن، ويوجد إدارة لمناطق ومدن الشحن، مع بيانات شحن على مستوى المنتج أو variant.

  • قائمة محافظات/حكومات للواجهة.
  • Shipment zones و cities مترجمة.
  • بيانات شحن المنتج: length، width، height، weight، min/max estimated delivery.
  • تكلفة شحن افتراضية من config، وتضاف للإجمالي النهائي.

12. العملاء والحسابات

Members

شرح المنتج: يوجد نظام members منفصل عن admin، مع OTP، تسجيل، دخول، وتحديث بيانات.

  • send verification OTP، verify OTP، register، login.
  • جلب بيانات المستخدم وتحديثها للمستخدم المسجل.
  • Admin يمكنه عرض العملاء، تفاصيل العميل، وطلبات العميل.
  • الطلبات تدعم مستخدم مسجل أو guest.

13. Dashboard Statistics

Analytics

شرح المنتج: إحصائيات تشغيلية للـ admin تساعده يشوف أداء المتجر بسرعة.

  • Overview يومي، أسبوعي، شهري، سنوي.
  • Recent orders ببيانات العميل والحالة والإجمالي.
  • نسب حالات الطلبات حسب status.
  • Best selling للمنتجات والـ variants والـ bundles مع quantity و revenue.
  • حساب profit شهري/سنوي من revenue ناقص cost و expenses.

14. صلاحيات الإدارة

Security

شرح المنتج: لوحة الإدارة محمية بـ Sanctum وصلاحيات roles/permissions، مع permission لكل عملية create/update/delete/view.

  • Admin login منفصل.
  • حماية routes بـ auth:sanctum.
  • صلاحيات عامة للـ CRUD وصلاحيات Ecommerce مثل cart، order، points، wishlist، statistics.
  • مناسب لفرق تشغيل متعددة: content manager، operations، sales admin، super admin.

15. CMS والمحتوى العام

CMS

شرح المنتج: بجانب المتجر، النظام يحتوي على modules لإدارة محتوى الموقع وصفحات الشركة.

  • Sliders، categories، brands، industries، pages، blogs، events، services، branches، locations.
  • About، contact، main contact، social links، media images/videos، FAQs، certificates، team، clients، feedback، achievements، our works.
  • Dynamic frontend endpoints لجلب محتوى، فلترة، gallery، search.
  • Meta settings وحقول SEO مترجمة لكثير من الكيانات.

16. الاستشارات والتدريب والطلبات الخارجية

Lead Capture

شرح المنتج: المنصة تدعم نماذج إضافية لجمع leads ورسائل خارج رحلة الشراء.

  • Contact/send-message.
  • Applicant store.
  • Consultation form.
  • Training form.
  • هذه الميزات تفيد في بيع المنتج لعملاء يقدمون خدمات أو تدريب بجانب المنتجات.

سيناريوهات بيع جاهزة

سيناريو 1: متجر منتجات بسيطة

نشرح أن العميل يقدر يبدأ بسرعة بمنتجات لها صور وسعر ومخزون وتصنيفات وخصومات، بدون الحاجة لتعقيد variants.

سيناريو 2: متجر ملابس أو منتجات لها أحجام

نركز على options و variants: كل لون/مقاس له SKU ومخزون وسعر وصور، والعميل في الواجهة يختار الاختيار المناسب.

سيناريو 3: متجر يريد عروض وباقات

نبيع bundles و promotions: باقات جاهزة، كوبونات، خصومات بنسبة أو قيمة ثابتة، وسقف خصم وحدود استخدام.

سيناريو 4: عميل يهتم بالتشغيل والمخزون

نركز على stock batches، order status، refund stock restore، statistics، best selling، profit monthly/annually.

سيناريو 5: شركة لديها محتوى وخدمات بجانب المتجر

نوضح أن المنتج ليس Ecommerce فقط، بل معه CMS لإدارة الصفحات والمدونة والفريق والفروع والاستشارات والتدريب.

ملاحظات مهمة قبل البيع النهائي

  • المشروع Backend/API وليس storefront UI أو admin UI كامل داخل هذا الريبو.
  • Payment gateway غير ظاهر كتكامل جاهز؛ الموجود هو `payment_method` و `payment_status` وإدارة حالة الدفع.
  • بعض أسماء routes والكود بها spelling قديم مثل `bundel` و `varaint` و `verfiy`؛ يفضل عدم ذكرها للعميل إلا في تسليم API documentation.
  • الاختبارات الآلية الموجودة قليلة، لذلك قبل إطلاق production ينصح بعمل test coverage للـ cart/order/stock/coupon/points.
  • الشحن حاليا يعتمد على government/address وتكلفة افتراضية من config، وأي قواعد شحن متقدمة تحتاج scope منفصل.

الخلاصة لفريق المبيعات

هذا المنتج مناسب كقاعدة قوية لمتجر إلكتروني قابل للتوسع. أقوى نقاطه: تعدد اللغات، دعم المنتجات البسيطة والـ variants، إدارة المخزون والطلبات، الباقات والعروض، الشراء كضيف أو مستخدم، loyalty points، وإدارة محتوى الموقع. عند العرض على العميل، ركز على أن المنصة تغطي التشغيل الكامل للمتجر وليس فقط عرض المنتجات.
